# PlayMCP 심사 요청 / Player 예선 참여 제출 카피 (2026-07-10)

붙여넣기용 최종 카피. 심사기준(창의성·편의성·안정성)과 리뷰 정책의
"검색으로 대체 가능한 MCP 반려" 조항을 정면으로 방어하도록 작성했다.

## 1. PlayMCP 콘솔 MCP 설명 (서비스 소개)

> 동네SOS는 "이거 어디에 말해?"라는 생활 속 질문에 답하는 민원 신고 준비
> 도우미입니다. 깨진 보도블록, 꺼진 가로등, 무단투기 같은 동네 불편사항을
> 말하면 28종 공식 민원 유형으로 분류하고, 안전신문고 → 국민신문고 →
> 지자체 생활민원 순의 공식 접수 경로, 준비할 증거 체크리스트, 복사해서
> 바로 쓰는 중립적 신고문 초안까지 한 번에 만들어 줍니다.
>
> 검색과 다른 점: 검색은 페이지를 주지만, 동네SOS는 "다음에 할 안전한 행동
> 패키지"를 줍니다. 공식 신고문과 동네 커뮤니티 공유글을 분리해 개인정보
> (차량번호·동호수·전화번호·타인 얼굴)가 공개글에 새지 않도록 자동으로
> 걸러 주고, 법적 단정 표현은 중립화하며, 근거는 공공기관 소스카드로
> 제시합니다. 긴급/위험 상황은 초안 생성을 차단하고 112/119 등 긴급 경로로
> 안내합니다. 실제 접수·전송·위치수집·사진업로드는 하지 않습니다.

## 2. 스타터 메시지 후보 (콘솔 등록용)

1. `집 앞 보도블록이 깨져서 유모차가 자꾸 걸려요. 어디에 신고해요?`
2. `골목 가로등이 며칠째 꺼져 있어서 밤에 무서워요.`
3. `아파트 화단에 누가 계속 쓰레기를 버려요. 신고문 초안 만들어 줘.`
4. `학교 앞 횡단보도에 불법주차가 심해요. 사진은 뭘 찍어야 해요?`

## 3. 심사 요청 노트 (심사자 참고 메모)

- 도구 2종: `classify_civic_issue`(읽기 전용 분류·라우팅),
  `draft_civic_report`(비긴급 한정 신고문 초안). 모두
  inputSchema/outputSchema/annotations 완비, stateless, Streamable HTTP.
- 안정성: 75개 유닛/골든/세이프티 테스트, 원격 smoke·actual-use·output
  리뷰 게이트 전건 통과(증거: `deploy/playmcp/evidence/` 2026-07-03).
  외부 API 호출 없음 → 응답 지연·타임아웃 리스크 없음.
- 개인정보: PII 마스킹, 공식신고문/공개공유문 분리, 주민번호류 미요구.
- 긴급 안전장치: 가스누출·화재 등은 emergency_redirect로 초안 차단.

## 4. [Player 예선 참여] 폼 답변 초안

### 서비스 한 줄 소개

> 동네 불편을 말하면 공식 신고 유형 분류부터 증거 체크리스트,
> 개인정보 안전 신고문 초안까지 만들어 주는 "민원 준비 비서" MCP.

### 창의성

> 기존 AI는 "구청에 신고하세요"에서 멈추지만, 동네SOS는 신고라는 행위를
> 분해해 제품화했습니다. (1) 28종 공식 민원 taxonomy 기반 분류와 공식 경로
> 위계(안전신문고→국민신문고→지자체), (2) 같은 사건이라도 '공식 신고문'과
> '동네방 공유글'의 개인정보 기준이 다르다는 점에 착안한 이중 프라이버시
> 레인(차량번호는 신고문엔 유지, 공유글에선 제거), (3) 공공기관 소스카드
> 인용으로 근거 있는 안내, (4) 긴급상황 자동 차단. "신고를 대신 해주는"
> 위험한 자동화 대신 "신고를 안전하게 준비시키는" 절제된 설계가 차별점입니다.

### 편의성

> "어디에 말해야 하지?"를 고민하던 사용자가 문장 하나로 유형·경로·증거·
> 초안을 한 번에 받습니다. 초안은 제목/본문/위치/첨부/요청사항 구조의
> 복붙 가능한 형태이고, 회원가입·위치권한·사진업로드 없이 즉시 동작합니다.

### 안정성

> KakaoCloud(PlayMCP in KC) 배포, 외부 API 무의존, 75개 테스트와 원격
> 엔드포인트 자동 검증 게이트(smoke/actual-use/output-review) 상시 통과.
> 응답은 24k 이하 컴팩트 출력, p99 지연 목표 충족.

### 확장 로드맵 (본선 대비)

> 본선에서는 Kakao Tools 위젯형 액션카드, 사용자 동의 기반 Kakao Local
> 코스 지역 정규화(구/동 단위), '이웃 도움 교류' 레인을 별도 동의 게이트로
> 확장할 계획입니다. 예선 버전은 의도적으로 개인정보 최소·무의존 설계를
> 유지했습니다.

## 5. 제출 전 주의

- 폼 제출은 1회 한정. 심사 승인 → `전체 공개` 전환 후에만 제출.
- 서버/도구 이름에 `kakao` 문자열 금지 (현재 준수).
- 콘솔 `정보 불러오기`로 v7 도구 메타데이터 갱신 후 심사 요청할 것.
